## 1. The big picture — this is now the "MISFITS DATABASE"

The app is no longer just the Tribe. It's a **multi-faction umbrella** ("Misfits Database") hosting
factions/sections. **8 factions** exist (`FACTIONS` in app.js), each a config *skin* (name, brand,
theme colour+bg, its own data sheet, its own doc tabs). Switching faction re-skins the whole app +
loads that faction's data. The **Tribe** (rust/amber, default) and **Brotherhood of Steel** (red)
have REAL data wired; the other 6 (Vault/Legion/Bazaar/Enclave/Unity/NCR) are unlinked → show a
themed "coming soon" roster until a contributor adds a sheet. Adding a faction = one config entry.

The database keeps reading **Google Sheets**. Docs are read from **Google Docs** (Tribe Lore /
Roleplay Guide). The docs are being migrated INTO a **wiki** — see T35.

## 6. Pointers
- Repo `dr-orac/oracular-db` (GitHub Pages). Read-only Sheets — NEVER write to `CONFIG.sheetId`.
- Verify: `python3 tools/preview.py` → browser-preview MCP → `python3 tools/selfcheck.py`.
- Conventions: imperative commit subject + why-body, **no AI-signal trailers**, no AI-gen trademarks
  on the public site. One task = one commit.
- **Preview gotcha:** `tools/preview.py` injects `window.YUMA_SHEET_OVERRIDE` (the tribe mirror), which
  wins over the faction sheet in `effectiveSheetId()`. To test a non-tribe faction's REAL data in
  preview, `delete window.YUMA_SHEET_OVERRIDE` then `applyFaction('brotherhood')`. Production has no
  override, so faction sheets load normally.
- **Doc images are large lazy base64** — hard to hydrate under programmatic scroll in headless; scroll
  gently or call `_figSet(img, _docHydrateCtx.images)`. They render fine in a real browser.
