## 1. What this is
A vanilla, no-build static **Misfits Database** — a Fallout/Pip-Boy terminal viewer for SS14 RP
factions. Deployed to GitHub Pages (repo `dr-orac/oracular-db`, live at
https://dr-orac.github.io/oracular-db/). One `index.html` + `styles.css` + `app.js` + self-hosted
fonts, no build step. Data: **Google Sheets** rosters + **Google Docs** lore + the **Misfits wiki**.

**Hard rules (unchanged):** never write to `CONFIG.sheetId` (the tribe's source sheet); NO AI-generated
signals anywhere (no Co-Authored-By trailers, no AI/LLM/agent language in tracked files or commits, no
AI-gen trademarks on the public site); one task = one commit (imperative subject + why-body); verify in
the preview + `python3 tools/selfcheck.py` clean before committing.

## 3. Key architecture (unchanged this session unless noted)
- `FACTIONS` config map + `FACTION_ORDER` + `FACTION_ICONS` + `FACTION_WIKI` (new). `activeFaction()`,
  `applyFaction()`, `factionAppearance()`, `fkey()`. Per-faction prefs under `mdb-<aspect>-<faction>`.
- Hash router: `#home` · `#<faction>/<section>[/<target>]` · **`#wiki/<Page>`** (top-level).
  `applyRoute`/`writeRoute`/`parseRoute`/`openPendingChar`/`scrollPendingAnchor`.
- Doc reader: `loadDoc`→`docClean` (strict whitelist; strips `<div>`). Wiki: `loadWiki`→`renderWiki`
  (structure-aware)→`wikiClean`/`docClean`. TOC via `buildDocSidebar` (from rendered headings).
- Frame modes: `body[data-frame="screen"|"border"]`; `data-frametint="olive"|"theme"`. Chassis is a
  self-contained module (styles.css, tagged @chassis-module).
- Preview: `python3 tools/preview.py` rebuilds `/tmp/yuma-live` + injects `MDB_SHEET_OVERRIDE`. **Preview
  gotcha:** a hash-only nav does NOT reload app.js — force `location.href=…?r=<ts>` for a fresh bundle.
  **Sandbox gotcha:** Google's gviz/Docs endpoints are network-blocked here (rosters/doc images won't
  load); only the wiki host is reachable. Verify Sheets/Docs-dependent work on the LIVE site.
