## 1. What this is
A vanilla, no-build static **Misfits Database** — a Fallout/Pip-Boy terminal viewer for SS14 RP
factions. Deployed to GitHub Pages (repo `dr-orac/oracular-db`, live at
https://dr-orac.github.io/oracular-db/). One `index.html` + `styles.css` + `app.js` + self-hosted
fonts, no build step. Data: **Google Sheets** rosters + **Google Docs** lore + the **Misfits wiki**.

**Hard rules (unchanged):** never write to `CONFIG.sheetId` (the tribe's source sheet); NO AI-generated
signals anywhere (no Co-Authored-By trailers, no AI/LLM/agent language in tracked files or commits, no
AI-gen trademarks on the public site); one task = one commit (imperative subject + why-body); verify in
the preview + `python3 tools/selfcheck.py` clean before committing.

## 3. Key architecture (deltas from prior handoffs in **bold**)
- `FACTIONS` config map + `FACTION_ORDER` + `FACTION_ICONS` + `FACTION_WIKI`. `activeFaction()`,
  `applyFaction()`, `factionAppearance()`, `fkey()`. Per-faction prefs under `mdb-<aspect>-<faction>`.
- **Two-level masthead**: row 1 (`.primary-nav`) = Home / Faction / Wiki + cog; row 2 (`#topnav`) =
  faction sections (**Roster · Relations · <docs>**). `renderPrimaryNav` / `renderNav` / `renderBrand`.
- Hash router: `#home` · `#<faction>/<section>[/<target>]` (**section ∈ roster·relations·<docid>**) ·
  `#wiki/<Page>`. `applyRoute`/`writeRoute`/`parseRoute`/`openPendingChar`/`scrollPendingAnchor`.
- `setSection` valid ids: home·roster·**relations**·wiki·<factionDoc>. Roster+Relations are roster-data-
  backed (`showRosterFor`/`showRelationsFor` share the same live-vs-coming-soon gate); docs → `loadDoc`;
  wiki → `loadWiki`.
- Doc reader: `loadDoc`→`docClean` (strict whitelist; strips `<div>`; **themes `<img>` as `.docfig`**).
  **Wiki: `loadWiki`→`normaliseWikiImages`→`renderWiki`→`wikiClean`/`docClean`.** TOC via `buildDocSidebar`.
- Frame modes: `body[data-frame="screen"|"border"]`; `data-frametint="olive"|"theme"`. Chassis is a
  self-contained module in styles.css (tagged @chassis-module).

## 4. Gotchas for the next maintainer (read before sandbox-testing)
- **Sandbox blocks Google.** gviz/Docs endpoints are network-blocked here, so rosters + doc images never
  load; the preview's live-mirror fetch is also INTERMITTENT (often hangs on "ACCESSING PIP-LINK").
  Anything roster-backed (roster list, cards, Relations) can only be verified with injected mock data or
  on the LIVE site. Only the wiki host (`wiki.misfitsystems.net`) is reliably reachable.
- **`window.state` is NOT the app's render state.** There is a separate global `window.state`, but the
  real state lives in a closure — setting `window.state.model` does nothing. To exercise roster-backed
  views in the browser you must let the live fetch succeed (flaky) or drive the real functions after it
  loads. (This burned a lot of time; don't repeat it.)
- **Preview reload:** `python3 tools/preview.py` rebuilds `/tmp/yuma-live`; a hash-only nav does NOT
  reload app.js — append `?r=<ts>` to force a fresh bundle.
- **The sandbox browser has `prefers-reduced-motion: reduce` ON.** So motion features (the CRT
  faction-switch transition, home-tile entrances, doc-loader, boot) do NOT play here — this is not a bug.
  To see an animation, scrub it with a paused `animation-delay` + a temporary `!important` override of the
  reduced-motion rule, or verify on a real machine.
- **The preview pane downscales wide viewports** (e.g. a 1360-wide viewport into an ~800px pane), so 1px
  lines / small type look fainter than reality — trust `getBoundingClientRect`/computed-style measurements
  over the screenshot for fine detail.
- Verify visual work in the preview + `python3 tools/selfcheck.py` clean (pre-commit hook runs it too).
